◆ NetworkingEventType

enum GameStudio.HunterGatherer.Networking.Events.NetworkingEventType : byte
strong

Defines the different types of networking events that can happen. Each has its own type of content to return

Enumerator
LobbyEnd 

Used when the lobby state ends. When sending, content should be null. When listening to this, the content will also be null

StatSystemModified 

Used when a client wants to modify another clients stat system. When sending or listening, content should be StatModificationInfo.

ItemSourceModified 

Used by the host to let clients know an item source has been modified. When sending or listening content is be ItemSourceModificationInfo

PlayerWinGame 

Used to notify a client it won the game. When sending or listening, content should an empty object

WeaponEquiped 

Used by a client to let the other clients know a unit aquipped a weapon. When sending or listening content should be ItemEquipInfo

ArmorEquiped 

Used by a client to let the other clients know a unit aquipped a weapon. When sending or listening content should be ArmorEquipInfo

SceneObjectSpawn 

Used when a client wants to instantiate a scene object but hasn't the rights to (isn't the host). Content is SceneObjectSpawnInfo

StaticObjectSpawn 

Used when a client wants to instantiate a static networked object. Content is SceneObjectSpawnInfo

StaticObjectDestroy 

Used when a client wants to destroy a static networked object. Content is the targets view id

InteractionInfoSpawn 

Used when a client wants to share interaction info. Content is InteractionInfoSpawnData

PackedItemEquiped 

Used by a client to let the other clients know a unit equipped a packed item. When sending or listening content should be PackedItemInfo

TentModified 

Used by the host to know if an tent has been occupied by x amount of units. When sending or listening content is be TentModificationInfo

UnitAnimated 

Used when a client wants to let others know his unit did an animation. Content is UnitAnimationInfo

Enumerator

Used when a client wants to pickup an item. The content needs to contain only the photonViewID of the pickupable item. the callback content will contain the player able to pickup

Used when a client wants to take an item from an item source and thus remove it, the content needs to contain only the photonViewID of the berry bush. the callback content will contain the player able to pickup
